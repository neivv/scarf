1//! Scarf is a library for analyzing x86 functions.
2//!
3//! The main concept of scarf is that the user gives [`FuncAnalysis`] a function
4//! which will be simulated. `FuncAnalysis` will walk through the function's different
5//! execution paths, while calling [user-defined callbacks](Analyzer), letting the user code
6//! to examine execution to extract information it needs. The callback is also able to
7//! modify state and have some control over execution, allowing scarf to be adapted for cases
8//! where the library is not quite able to handle unusual assmebly patterns.
9//!
10//! Examples of problems that could be answered with scarf:
11//! - Find all child function calls of a function, where one of the arguments is
12//!     constant integer between 0x600 and 0x700
13//! - If the function writes a 64-bit value to `(Base pointer)+0x28`, return the base pointer
14//!     and value which was written to. That is, detect writes to a field of a struct when the
15//!     field offset is known to be 0x28.
16//! - Check if the function reads memory at given constant address, and track non-stack
17//!     locations where the read value is passed to.
18//! - Determine all constant arguments that are passed to a certain function `f`, by analyzing all
19//!     of the functions calling `f`.
20//! - Find a point where the function compares some value `x` to be less than constant 0x100, and
21//!     return what expression `x` is, as well as the jump address and whether it has to be
22//!     changed to always or never to jump in order to always go to `x < 0x100` branch.
23//!
24//! In general, scarf is still relatively low-level in its execution representation.
25//! Good analysis results often require user to handle edge cases, which often requires
26//! iterative improvements to analysis code when you come across an executable that the
27//! analysis quite does not work on. As ultimately the only input to scarf analysis is often
28//! just the executable binary, keeping tests using those binaries to prevent scarf-using code
29//! from suddenly regressing is a good idea.
30//!
31//! Scarf strives to be fast enough to analyze an average function in less than 1 millisecond
32//! even on slower machines. This makes it quite feasible to brute force every function of
33//! even in larger executable in few minutes, as well as have more targeted analysis be fast
34//! enough that it can be ran without anyone noticing. Some of this speed means giving up
35//! accuracy, and if an adversary codegen wanted to explicitly break scarf, it would likely
36//! at least require user callback to actively help scarf from breaking.
37//! The simulation accuracy issues do not seem to cause too much problem in regular
38//! compiler-generated code though.
39//!
40//! The following are main types used by scarf:
41//!
42//! - [`FuncAnalysis`] - The entry point for scarf analysis. Walks through and keeps track of
43//!     all branches the execution may take.
44//! - [`trait Analyzer`](Analyzer) - User-implemented trait that `FuncAnalysis` calls back to,
45//!     allowing user code to query and manipulate analysis.
46//! - [`analysis::Control`] - A type passed to `Analyzer` callbacks, providing various
47//!     ways to query and manipulate analysis state.
48//! - [`BinaryFile`] - Contains sections of the binary, including code that is to be simulated.
49//!     - [`BinarySection`] - A single section, practically just `Vec<u8>` and a base address.
50//! - [`VirtualAddress32`] / [`VirtualAddress64`] - Integer newtype representing a constant
51//!     address, usually in `BinaryFile`
52//!     - [`trait exec_state::VirtualAddress`](exec_state::VirtualAddress) - A trait allowing
53//!     handling both address sizes generically.
54//! - [`Operand`] - The main value / expression type of scarf.
55//! - [`OperandContext`] - Allocation arena and interner for `Operand`s. Has to outlive
56//!     `FuncAnalysis`, so user code is required to create this and pass to rest of scarf.
57//! - [`trait ExecutionState`](exec_state::ExecutionState) - Holds all of the simulated
58//!     CPU and memory state of one point in analysis's execution.
59//!     Concrete types are [`ExecutionStateX86`] and [`ExecutionStateX86_64`].
60

142/// `VirtualAddress32` represents a constant 32-bit memory address.
143///
144/// See also [`exec_state::VirtualAddress`] trait for the trait
145/// is used by generic code to operate on either this type or
146/// 64-bit [`VirtualAddress64`].
147///
148/// Most of scarf does not require memory addresses be a integer constant,
149/// using [`Operand`s](Operand) instead. `VirtualAddress` is mainly used when
150/// reading from [`BinaryFile`], and when creating new [`analysis::FuncAnalysis`].
151///
152/// The only notable difference from a plain `u32` is that
153/// addition and subtraction(*) are defined to wrap on overflow.

156/// overflow checks are enabled, while subtracting `VirtualAddress32 - u32` wraps.

334/// Contains the binary that is to be analyzed, loaded to memory.
335///
336/// Not much more than a Vec<[BinarySection]>, and functions to read
337/// the correct section in different ways, when given
338/// [`VirtualAddress`](exec_state::VirtualAddress).
339///
340/// `BinaryFile` is often passed as by reference `&'e BinaryFile`, using same lifetime as
341/// [`OperandContext`]. This would not be required, as `BinaryFile` does not hold any references
342/// to `OperandContext`, the lifetime was chosen to be same to keep all code interfacing
343/// with scarf simpler, only requiring a single lifetime being passed around. This unfortunately
344/// does require `BinaryFile` to be created so that it outlives the `OperandContext` passed
345/// to [`FuncAnalysis`], which usually should not be a too big of a problem.
346///
347/// [`parse`] and [`parse_x86_64`] can be used to load a `BinaryFile` from Windows
348/// PE executable, [`raw_bin`] can be used to create a `BinaryFile` from arbitrary
349/// sections.
350#[derive(Debug, Clone)]
351pub struct BinaryFile<Va: exec_state::VirtualAddress> {
352    pub base: Va,
353    sections: Vec<BinarySection<Va>>,
354    relocs: Vec<Va>,
355}
356
357/// Single section of a [`BinaryFile`].
358#[derive(Debug, Clone)]
359pub struct BinarySection<Va: exec_state::VirtualAddress> {
360    pub name: [u8; 8],
361    pub virtual_address: Va,
362    pub virtual_size: u32,
363    pub data: Vec<u8>,
364}

374impl<Va: exec_state::VirtualAddress> BinaryFile<Va> {
375    #[inline]
376    pub fn base(&self) -> Va {
377        self.base
378    }
379
380    /// Returns `(address - self.base()) as u32`.
381    ///
382    /// Panics on overflow when overflow checks are enabled.
383    /// If the address is known to be in one of the sections,
384    /// at least on Windows any PE file will have offset that should fit in u32
385    /// even if truncated.
386    pub fn rva_32(&self, address: Va) -> u32 {
387        (address.as_u64() as u32) - (self.base.as_u64() as u32)
388    }
389
390    /// Returns `(address - self.base()) as u32`, or `None` if `address`
391    /// is less than `self.base()` or the difference does not fit in `u32`.
392    pub fn try_rva_32(&self, address: Va) -> Option<u32> {
393        u32::try_from((address.as_u64()).checked_sub(self.base.as_u64())?).ok()
394    }
395
396    /// Returns `(address - self.base()) as u64`.
397    ///
398    /// Panics on overflow when overflow checks are enabled.
399    pub fn rva_64(&self, address: Va) -> u64 {
400        address.as_u64() - self.base.as_u64()
401    }
402
403    /// Returns `(address - self.base()) as u64`, or `None` if `address`
404    /// is less than `self.base()`.
405    pub fn try_rva_64(&self, address: Va) -> Option<u64> {
406        address.as_u64().checked_sub(self.base.as_u64())
407    }
408
409    /// Returns the section named ".text".
410    ///
411    /// Panics if this section for some reason does not exist.
412    ///
413    /// `[code_sections]` should be preferred if the user code wants to handle possibility
414    /// of multiple executable sections... But currently its implementation is bad and
415    /// just returns section named ".text", if it exists...
416    pub fn code_section(&self) -> &BinarySection<Va> {
417        self.section(b".text\0\0\0").unwrap()
418    }
419
420    /// Returns iterator, which yields all `[BinarySection]`s of this `BinaryFile`.
421    pub fn sections<'a>(&'a self) -> impl Iterator<Item = &'a BinarySection<Va>> + 'a {
422        self.sections.iter()
423    }
424
425    pub fn code_sections<'a>(&'a self) -> impl Iterator<Item = &'a BinarySection<Va>> + 'a {
426        self.sections.iter().filter(|x| &x.name[..] == &b".text\0\0\0"[..])
427    }
428
429    /// Returns section by name, if it exists.
430    pub fn section(&self, name: &[u8; 0x8]) -> Option<&BinarySection<Va>> {
431        self.sections.iter().find(|x| x.name[..] == name[..])
432    }
433
434    /// Returns a section containing the address, if it exists
435    pub fn section_by_addr(&self, address: Va) -> Option<&BinarySection<Va>> {
436        self.sections.iter().find(|x| {
437            address >= x.virtual_address && address < (x.virtual_address + x.data.len() as u32)
438        })
439    }
440
441    /// Range is relative from base
442    pub fn slice_from(&self, range: std::ops::Range<u32>) -> Result<&[u8], OutOfBounds> {
443        self.slice_from_address(self.base + range.start, range.end - range.start)
444    }
445
446    /// Receives a slice of data from address with length `len`, or Err(OutOfBounds)
447    /// if there is no single section containing the entire slice.
448    pub fn slice_from_address(&self, start: Va, len: u32) -> Result<&[u8], OutOfBounds> {
449        self.section_by_addr(start)
450            .and_then(|s| {
451                let section_relative = start.as_u64() - s.virtual_address.as_u64();
452                s.data.get(
453                    section_relative as usize ..
454                    (section_relative + len as u64) as usize,
455                )
456            })
457            .ok_or_else(|| OutOfBounds)
458    }
459
460    /// Receives a slice of data from address to end of the respective section.
461    pub fn slice_from_address_to_end(&self, start: Va) -> Result<&[u8], OutOfBounds> {
462        self.section_by_addr(start)
463            .and_then(|s| {
464                let section_relative = start.as_u64() - s.virtual_address.as_u64();
465                s.data.get((section_relative as usize)..)
466            })
467            .ok_or_else(|| OutOfBounds)
468    }

470    /// Searches for section containing `addr`, and reads a little-endian u8 from there.
471    pub fn read_u8(&self, addr: Va) -> Result<u8, OutOfBounds> {
472        use crate::light_byteorder::ReadLittleEndian;
473        self.section_by_addr(addr)
474            .and_then(|s| {
475                let section_relative = addr.as_u64() - s.virtual_address.as_u64();
476                s.data.get(section_relative as usize..)
477            })
478            .and_then(|mut data| {
479                data.read_u8().ok()
480            })
481            .ok_or_else(|| OutOfBounds)
482    }

526    /// Searches for section containing `addr`, and reads a `VirtualAddress`
527    /// (Same size as the addresses of this `BinaryFile` are) from there.
528    pub fn read_address(&self, addr: Va) -> Result<Va, OutOfBounds> {
529        match Va::SIZE {
530            4 => self.read_u32(addr).map(|x| Va::from_u64(x as u64)),
531            8 => self.read_u64(addr).map(|x| Va::from_u64(x)),
532            x => panic!("Unsupported VirtualAddress size {}", x),
533        }
534    }

541impl<Va: exec_state::VirtualAddress> BinarySection<Va> {
542    /// Returns true if `address` is within this section.
543    #[inline]
544    pub fn contains(&self, address: Va) -> bool {
545        address >= self.virtual_address && address < self.end()
546    }
547
548    /// Returns end address (First byte not included) of this section.
549    #[inline]
550    pub fn end(&self) -> Va {
551        self.virtual_address + self.virtual_size
552    }
553
554    /// Receives a slice of data from address with length of `length, or None,
555    /// if the slice is not fully contained in the section.
556    pub fn slice_from_address(&self, address: Va, length: u32) -> Option<&[u8]> {
557        let offset = usize::try_from(
558            address.as_u64().wrapping_sub(self.virtual_address.as_u64())
559        ).ok()?;
560        self.data.get(offset..)?.get(..(length as usize))
561    }
562
563    /// Receives a slice of data from address to end of section, or None,
564    /// if the address is not in the section
565    pub fn slice_from_address_to_end(&self, address: Va) -> Option<&[u8]> {
566        let offset = usize::try_from(
567            address.as_u64().wrapping_sub(self.virtual_address.as_u64())
568        ).ok()?;
569        self.data.get(offset..)
570    }

595
596    /// Reads a little-endian `VirtualAddress`
597    /// (Same size as the addresses of this `BinarySection` are)
598    /// from `addr` if the address is within this section.
599    #[inline]
600    pub fn read_address(&self, addr: Va) -> Option<Va> {
601        match Va::SIZE {
602            4 => self.read_u32(addr).map(|x| Va::from_u64(x as u64)),
603            8 => self.read_u64(addr).map(|x| Va::from_u64(x)),
604            x => panic!("Unsupported VirtualAddress size {}", x),
605        }
606    }
607}

609impl<'e, Va: exec_state::VirtualAddress> BinaryFileWithCachedSection<'e, Va> {
610    /// Panics if given BinaryFile with no sections.
611    /// (Would be nice to have it just start referencing a static
612    /// dummy section, but it's bit pain to do with generics)
613    pub fn new(file: &'e BinaryFile<Va>) -> Self {
614        Self::try_new(file).expect("Empty BinaryFile")
615    }
616
617    pub fn try_new(file: &'e BinaryFile<Va>) -> Option<Self> {
618        Some(Self {
619            file,
620            last_section: file.sections().next()?,
621        })
622    }
623
624    #[cold]
625    fn update_last_section(&mut self, address: Va) -> Option<()> {
626        let new_section = self.file.section_by_addr(address)?;
627        if std::ptr::eq(new_section, self.last_section) {
628            None
629        } else {
630            self.last_section = new_section;
631            Some(())
632        }
633    }
634
635    pub fn read_u8(&mut self, address: Va) -> Option<u8> {
636        loop {
637            match self.last_section.read_u8(address) {
638                Some(s) => return Some(s),
639                None => self.update_last_section(address)?,
640            }
641        }
642    }
643
644    pub fn read_u16(&mut self, address: Va) -> Option<u16> {
645        loop {
646            match self.last_section.read_u16(address) {
647                Some(s) => return Some(s),
648                None => self.update_last_section(address)?,
649            }
650        }
651    }
652
653    pub fn read_u32(&mut self, address: Va) -> Option<u32> {
654        loop {
655            match self.last_section.read_u32(address) {
656                Some(s) => return Some(s),
657                None => self.update_last_section(address)?,
658            }
659        }
660    }
661
662    pub fn read_u64(&mut self, address: Va) -> Option<u64> {
663        loop {
664            match self.last_section.read_u64(address) {
665                Some(s) => return Some(s),
666                None => self.update_last_section(address)?,
667            }
668        }
669    }
670}

684/// Creates a `BinaryFile` from 32-bit Windows executable at `filename`.
685pub fn parse(filename: &OsStr) -> Result<BinaryFile<VirtualAddress32>, Error> {
686    use byteorder::{ByteOrder, LittleEndian};
687    use crate::Error::*;
688    let mut file = BufReader::new(File::open(filename)?);
689    let mut buffer = [0u8; 0x58];
690    let mut seek_pos = 0u64;
691    file.read_exact(&mut buffer[..0x40])?;
692    seek_pos += 0x40;
693    if LittleEndian::read_u16(&buffer[0..]) != 0x5a4d {
694        return Err(InvalidPeFile("Missing DOS magic"));
695    }
696    let pe_offset = LittleEndian::read_u32(&buffer[0x3c..]) as u64;
697    file.seek_relative((pe_offset as u64).wrapping_sub(seek_pos) as i64)?;
698    seek_pos = pe_offset as u64;
699    file.read_exact(&mut buffer)?;
700    seek_pos = seek_pos.wrapping_add(buffer.len() as u64);
701
702    if LittleEndian::read_u32(&buffer[0..]) != 0x0000_4550 {
703        return Err(InvalidPeFile("Missing PE magic"));
704    }
705
706    let section_count = LittleEndian::read_u16(&buffer[0x6..]);
707    let base = VirtualAddress32(LittleEndian::read_u32(&buffer[0x34..]));
708    let header_block_size = LittleEndian::read_u32(&buffer[0x54..]);
709
710    let mut sections = Vec::with_capacity(section_count as usize + 1);
711    // Read header
712    file.seek_relative(0u64.wrapping_sub(seek_pos) as i64)?;
713    let mut header_data = Vec::with_capacity(header_block_size as usize);
714    (&mut file).take(header_block_size as u64).read_to_end(&mut header_data)?;
715
716    let section_headers = match header_data.get(((pe_offset + 0xf8) as usize)..)
717        .and_then(|x| x.get(..(0x28 * section_count as usize)))
718    {
719        Some(s) => s,
720        None => return Err(InvalidPeFile("Sections not in header")),
721    };
722
723    // Read sections without extra buffering.
724    // This may not even be worth it as BufReader is smart enough to not
725    // use buffer for reads larger than the buffer, and small sections
726    // could be read with single syscall if using BufReader.
727    let mut file = file.into_inner();
728    // No guarantee of inner seek pos atm, set to invalid
729    seek_pos = u64::MAX;
730    for section in section_headers.chunks_exact(0x28) {
731        let virtual_size = LittleEndian::read_u32(&section[0x8..]);
732        let rva = LittleEndian::read_u32(&section[0xc..]);
733        let phys_size = LittleEndian::read_u32(&section[0x10..]);
734        let phys = LittleEndian::read_u32(&section[0x14..]);
735        if phys as u64 != seek_pos {
736            file.seek(io::SeekFrom::Start(phys as u64))?;
737        }
738        let mut data = Vec::with_capacity(phys_size as usize);
739        (&mut file).take(phys_size as u64).read_to_end(&mut data)?;
740        seek_pos = phys as u64 + phys_size as u64;
741        sections.push(BinarySection {
742            name: section[..8].try_into().unwrap(),
743            virtual_address: base + rva,
744            virtual_size,
745            data,
746        })
747    }
748
749    sections.push(BinarySection {
750        name: *b"(header)",
751        virtual_address: base,
752        virtual_size: header_block_size,
753        data: header_data,
754    });
755    Ok(BinaryFile {
756        base,
757        sections,
758        relocs: Vec::new(),
759    })
760}
