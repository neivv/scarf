41/// `Operand` is the type of values in scarf.
42///
43/// It is an immutable reference type, free to copy and expected to be passed by value.
44///
45/// Different types of `Operand`s are listed in [`OperandType`] enum.
46/// The main types of interest are:
47///
48/// - Single variables, such as `OperandType::Arch`, `OperandType::Custom`.
49/// - Constant integers, `OperandType::Constant`.
50/// - Expressions, `OperandType::Arithmetic`, using `Operand` as inputs for the expression,
51/// and as such, able to have arbitrarily deep tree of subexpressions making up the `Operand`.
52/// - Memory, `OperandType::Memory`, which is able to have any `Operand` representing the
53/// address.
54///
55/// All `Operand`s are created through [`OperandContext`] arena, which interns the created
56/// `Operand`s, allowing implementing equality comparison as a single reference equality
57/// check. The lifetime `'e` refers to this backing memory in `OperandContext`
58/// which the `Operand` reference points to.
59///
60/// # Simplification
61///
62/// Every created `Operand` which contains an expression is immediately simplified to a form
63/// that scarf considers to be the simplest equivalent form of the expression.
64///
65/// For example, subtracting any `Operand` from itself is simplified to zero, while adding
66/// any `Operand` to itself gets simplfied to multiplication by two:
67///
68/// ```rust
69/// let ctx = &scarf::OperandContext::new();
70/// // Create an `Operand` representing `rcx` register.
71/// // x86-64 registers are ordered as 'rax rcx rdx rbx  rsp rbp rsi rdi  r8 r9 r10...
72/// // As such, index of rcx is 1.
73/// let rcx = ctx.register(1);
74/// // Create an `Operand` representing `rcx - rcx` expression.
75/// let sub_result = ctx.sub(rcx, rcx);
76/// // Create an `Operand` representing constant 0.
77/// let zero = ctx.constant(0);
78/// // Subtraction result is also zero
79/// assert_eq!(sub_result, zero);
80///
81/// // And `rcx + rcx` is same as `rcx * 2`
82/// let add_result = ctx.add(rcx, rcx);
83/// // Most arithmetic `Operand` creation functions have a variation
84/// // which should be preferred when one of the operands is known to
85/// // be constant.
86/// // This is more concise and better for performance than the equivalent
87/// // `ctx.mul(rcx, ctx.constant(2))`.
88/// let mul_result = ctx.mul_const(rcx, 2);
89/// assert_eq!(add_result, mul_result);
90///
91/// // We can also have an `Operand` expression for equivalency:
92/// // `ArithOpType::Equal` evaluates to either 0 or 1 for the result.
93/// let eq_result = ctx.eq(add_result, mul_result);
94/// // Equivalent to `ctx.constant(1)`, but effectively free to evaluate.
95/// let one = ctx.const_1();
96/// assert_eq!(eq_result, one);
97/// ```
98///
99/// All `Operand`s, and as such, arithmetic expressions of the `Operand`s are considered
100/// 64-bit values. In general, arithmetic expressions wrap on overflow, and arithmetic smaller
101/// than 64-bit is represented by doing bitwise AND to only keep the low bits.
102///
103/// **Note:** Overflowing shifts (Shifting by more than 64 bits) are defined to result in
104/// zero, instead of the behaviour of x86 shift and rotate instructions on general-purpose
105/// registers, where shift count is taken as modulo 64. Scarf represents those instructions
106/// as `rax << (count & 63)`, and users generally do not have to think about this.
107///
108/// If scarf is given excessively complex arithmetic to simplify -- which is often
109/// seen in highly unrolled hash functions, scarf rather gives up than never finishes the
110/// simplification.
111///
112/// ## Canonicalization guarantees
113///
114/// Simplification tries to convert any equivalent expression to a single 'canonical' form,
115/// to make equality comparisons useful. As the simplification rules get improved, this may
116/// result in user code which matches against a specific form of arithmetic expression to break.
117///
118/// This does mean that updating scarf dependency can bring surprising breaking changes! Try to
119/// have tests on any code that has to match on `Operand`s.
120///
121/// The following canonicalizations are guaranteed to be stable across scarf changes,
122/// to allow code to be simpler and faster by not having to consider expression forms that are
123/// never created.
124///
125/// - Any chain of an operation, where the operation is commutative (Swapping left/right does
126/// not affect result; Mul/And/Or/Xor) will be simplified to have single operand on right
127/// and rest of the chain on left:
128///
129///     `(((a ^ b) ^ c) ^ d) ^ e`.
130/// - Expressions chains of multiple combined Add/Sub are simplified similar to above rule:
131///
132///     `(((a + b) - c) + d) + e`
133///
134///     No guarantee how adds and subs are ordered within the chain.
135/// - If any of the above commutative/add-sub chains has a constant, it is guaranteed to be
136/// the outermost right operand:
137///
138///     `(a & b) & ffff`
139/// - Equality expressions containing a non-zero constant always have the constant alone on right,
140/// with rest of the expression shuffled to left. If the constant is zero, and there is no
141/// subtraction on left side, the zero is also guaranteed to be on left.
142///
143///     That is, `x - y == 0` is simplified to either `x == y` or `y == x` instead,
144///     while `0 == x` is guaranteed be simplified to `x == 0`.
145///
146/// Consider using [`Operand::either`] for cases where the result you want may be on either
147/// left- or right-hand operand of the expression.
148///
149/// # Matching
150///
151/// The most comprehensive way to match against different values of `Operand` is to
152/// match on the [`OperandType`] enum, returned from [`Operand::ty`].
153///
154/// However, most of the time this leads to quite verbose code, especially when you want to
155/// try match against just a single variant of arithmetic expression or constant.
156///
157/// `Operand` includes a large amount of matching methods, starting with `if_`, such as
158/// [`if_constant`](Self::if_constant), returning `Option<u64>`,
159/// [`if_arithmetic_add`](Self::if_arithmetic_add), returning
160/// `Option<(Operand<'e>, Operand<'e>)>`, [`if_memory`](Self::if_memory),
161/// returning `Option<&'e MemAccess<'e>>`.
162///
163/// Using the matching methods is usually a lot more concise than using `match` on
164/// `OperandType`.
165///
166/// Note that the memory matching methods effectively have three tiers:
167/// - [`if_memory`](Self::if_memory), returning [`MemAccess`]
168///     if the `Operand` is memory operand at all.
169/// - [`if_mem64`](Self::if_mem64), [`if_mem32`](Self::if_mem32),
170///     and others, which still return `MemAccess`, but check
171///     that `mem.size` is of expected size
172/// - [`if_mem64_offset`](Self::if_mem64_offset), [`if_mem32_offset`](Self::if_mem32_offset),
173///     and others, which check both `MemAccess` size.
174///     and specific constant offset, commonly useful when matching on a structure field.
175///
176///     They return `Operand` for the base memory address.
177#[derive(Copy, Clone)]
178#[cfg_attr(feature = "serde", derive(Serialize), serde(transparent))]
179pub struct Operand<'e>(&'e OperandBase<'e>, PhantomData<&'e mut &'e ()>);

187#[cfg_attr(feature = "serde", derive(Serialize))]
188pub(crate) struct OperandBase<'e> {
189    ty: OperandType<'e>,
190    #[cfg_attr(feature = "serde", serde(skip_serializing))]
191    relevant_bits: Range<u8>,
192    #[cfg_attr(feature = "serde", serde(skip_serializing))]
193    flags: u8,
194    /// Bits that are used to avoid some comparisons when checking against OperandType
195    /// that has a subenum that gets commonly checked too.
196    /// 0xff00 = Tag; None, Memory, Arithmetic, Register
197    /// 0xff = Data; MemAccessSize for Memory, ArithOpType for Arithmetic
198    #[cfg_attr(feature = "serde", serde(skip_serializing))]
199    type_alt_tag: u16,
200    /// Used to implement Ord in almost-always-constant time.
201    ///
202    /// - For OperandType variants that fit in u64,
203    ///     this variable just holds the same data as the variant
204    ///     that can be used to compare this.
205    /// - For Arithmetic / ArithmeticFloat, this has high 5 bits set to ArithOpType
206    ///     and rest are hash (stable across executions) of sort_order of the
207    ///     left/right ops (ArithmeticFloat size is not included at all)
208    /// - For Memory / SignExtend, this holds discriminant and sort_order
209    ///     (Shifted right to fit in discriminant) of `base`
210    ///     (Offset / size are not included)
211    /// The values are picked so that there wouldn't be too many conflicts
212    /// in average sort case, and that the sort order is 'sensible' for most
213    /// operands (Registers sorted by their index, constants by constant value)
214    /// And when there are conflicts (E.g. Mem32[rax], Mem8[rax], Mem32[rax + 8]
215    /// have all same sort_order) it should be quick to check remaining variables.
216    /// (In general you'd never compare two equal `OperandBase`s, since `Operand`
217    /// comparison includes equality check already.)
218    ///
219    /// `sort_order` could be also used as input to hash, though it would require
220    /// remembering to add data that is not included, with `Memory` and other
221    /// large types.
222    #[cfg_attr(feature = "serde", serde(skip_serializing))]
223    sort_order: u64,
224    #[cfg_attr(feature = "serde", serde(skip_serializing))]
225    relevant_bits_mask: u64,
226}

275impl<'e> Ord for Operand<'e> {
276    fn cmp(&self, other: &Operand<'e>) -> Ordering {
277        if ptr::eq(self.0, other.0) {
278            Ordering::Equal
279        } else {
280            let OperandBase {
281                ref ty,
282                relevant_bits: _,
283                flags: _,
284                type_alt_tag: _,
285                sort_order,
286                relevant_bits_mask: _,
287            } = *self.0;
288
289            let other_ty = &other.0.ty;
290            let self_discr = ty.discriminant();
291            let other_discr = other_ty.discriminant();
292            let order = self_discr.cmp(&other_discr);
293            if order != Ordering::Equal {
294                return order;
295            }
296
297            let order = sort_order.cmp(&other.0.sort_order);
298            if order != Ordering::Equal {
299                return order;
300            }
301            ty.cmp_sort_order_was_eq(other_ty)
302        }
303    }
304}

306impl<'e> PartialOrd for Operand<'e> {
307    fn partial_cmp(&self, other: &Operand<'e>) -> Option<Ordering> {
308        Some(self.cmp(other))
309    }
310
311    // Explicit lt implementation is bit faster for sorting which uses it
312    fn lt(&self, other: &Operand<'e>) -> bool {
313        let OperandBase {
314            ref ty,
315            relevant_bits: _,
316            type_alt_tag: _,
317            flags: _,
318            sort_order,
319            relevant_bits_mask: _,
320        } = *self.0;
321
322        let other_ty = &other.0.ty;
323        let self_discr = ty.discriminant();
324        let other_discr = other_ty.discriminant();
325        if self_discr != other_discr {
326            return self_discr < other_discr;
327        }
328
329        if sort_order != other.0.sort_order {
330            return sort_order < other.0.sort_order;
331        }
332
333        if ptr::eq(self.0, other.0) {
334            return false;
335        }
336        matches!(ty.cmp_sort_order_was_eq(other_ty), Ordering::Less)
337    }
338}

383fn display_operand<'e>(f: &mut fmt::Formatter, op: Operand<'e>, recurse: u8) -> fmt::Result {
384    use self::ArithOpType::*;
385    if recurse >= 20 {
386        match *op.ty() {
387            OperandType::Memory(..) | OperandType::Arithmetic(..) |
388                OperandType::ArithmeticFloat(..) | OperandType::SignExtend(..) |
389                OperandType::Select(..) =>
390            {
391                return write!(f, "(...)");
392            }
393            _ => (),
394        }
395    }
396    match *op.ty() {
397        OperandType::Arch(r) => {
398            if let Some(fmt) = r.arch_def.fmt {
399                fmt(f, r.value())
400            } else {
401                write!(f, "Arch({:x})", r.value())
402            }
403        }
404        OperandType::Constant(c) => write!(f, "{:x}", c),
405        OperandType::Memory(ref mem) => {
406            let (base, offset) = mem.address();
407            if let Some(c) = mem.if_constant_address() {
408                write!(f, "Mem{}[{:x}]", mem.size.bits(), c)
409            } else {
410                write!(f, "Mem{}[", mem.size.bits())?;
411                display_operand(f, base, recurse + 1)?;
412                if offset != 0 {
413                    let (sign, offset) = match offset < 0x8000_0000_0000_0000 {
414                        true => ('+', offset),
415                        false => ('-', 0u64.wrapping_sub(offset)),
416                    };
417                    write!(f, " {} {:x}]", sign, offset)
418                } else {
419                    write!(f, "]")
420                }
421            }
422        }
423        OperandType::Undefined(id) => write!(f, "Undefined_{:x}", id.0),
424        OperandType::Arithmetic(ref arith) | OperandType::ArithmeticFloat(ref arith, _) => {
425            let l = arith.left;
426            let r = arith.right;
427            let (prefix, middle) = match arith.ty {
428                Add => ("(", " + "),
429                Sub => ("(", " - "),
430                Mul => ("(", " * "),
431                Div => ("(", " / "),
432                Modulo => ("(", " % "),
433                And => ("(", " & "),
434                Or => ("(", " | "),
435                Xor => ("(", " ^ "),
436                Lsh => ("(", " << "),
437                Rsh => ("(", " >> "),
438                Equal => ("(", " == "),
439                GreaterThan => ("(", " > "),
440                SignedMul => ("mul_signed(", ", "),
441                MulHigh => ("mul_high(", ", "),
442                ToFloat => ("to_float(", ""),
443                ToDouble => ("to_double(", ""),
444                ToInt => ("to_int(", ""),
445            };
446            write!(f, "{}", prefix)?;
447            display_operand(f, l, recurse + 1)?;
448            if middle != "" {
449                write!(f, "{}", middle)?;
450                display_operand(f, r, recurse + 1)?;
451            }
452            write!(f, ")")?;
453            match *op.ty() {
454                OperandType::ArithmeticFloat(_, size) => {
455                    write!(f, "[f{}]", size.bits())?;
456                }
457                _ => (),
458            }
459            Ok(())
460        },
461        OperandType::SignExtend(val, ref from, ref to) => {
462            write!(f, "signext_{}_to_{}(", from.bits(), to.bits())?;
463            display_operand(f, val, recurse + 1)?;
464            write!(f, ")")
465        }
466        OperandType::Select(condition, yes, no) => {
467            // select(condition ? yes : no)
468            write!(f, "select(")?;
469            for pair in [
470                (condition, " ? "),
471                (yes, " : "),
472                (no, ")"),
473            ] {
474                display_operand(f, pair.0, recurse + 1)?;
475                write!(f, "{}", pair.1)?;
476            }
477            Ok(())
478        }
479        OperandType::Custom(val) => {
480            write!(f, "Custom_{:x}", val)
481        }
482    }
483}

698/// An arena for allocating and interning [`Operand`]s.
699///
700/// Usually the type alias to reference [`OperandCtx`] is used instead.
701/// A variable of type `OperandContext` / `OperandCtx` is usually named `ctx`.
702///
703/// `OperandContext` is used to create `Operand`s, returning reference to already existing
704/// `Operand` if it was already created, allowing equality comparisons be done with a simple
705/// pointer equality check.
706///
707/// Additionally `OperandContext` makes sure that all created `Operand`s are simplified and
708/// canonicalized. For example, `rcx - rcx` is simplified to a constant `0`, while `rax + rcx`
709/// and `rcx + rax` return references to the same `Operand`. While the choice between
710/// `rax + rcx` or `rcx + rax` for canonical form is not specified, some canonicalizations
711/// [are guaranteed](./struct.Operand.html#canonicalization-guarantees),
712/// and user code may rely on them to simplify matching of `Operand`s.
713///
714/// Most of the functions of `OperandContext` are for creating an [`Operand`] with specific
715/// [`OperandType`], though there is also a [`mem_access`](Self::mem_access) to create
716/// [`MemAccess`] (As it also has canonicalization requirements), and few general-use
717/// functions
718///
719/// # `OperandContext` lifetime `'e`
720///
721/// The `'e` lifetime passed around everywhere in scarf ultimately refers to `Operand`s, which
722/// are a reference to `&'e OperandContext<'e>`, usually shortened with a type alias to
723/// [`OperandCtx<'e>`].
724///
725/// `'e` is an invariant lifetime, which generally makes it impossible to mix allocations from
726/// two different `OperandContext`s. However, if you have `OperandCtx<'static>` or create two
727/// separate `OperandContext`s in same scope, Rust allows assigning the same lifetime to them,
728/// allowing the allocations to be mixed. While this is not expected to cause memory safety
729/// issues, it will make scarf produce nonsensical results as equality comparisons are no
730/// longer reliable. Try to avoid cases where there are two `OperandContext` with same lifetime.
731///
732/// A common use case for multiple `OperandContext`s is to use a shorter-lived one to analyze
733/// functions (Which can allocate tens or hundrerds of thousands `Operand`s total), that will
734/// be throwns away to release memory afterwards, and a longer-lived `OperandContext` to hold
735/// `Operand` results that you are interested in.
736/// `Operand`s can be copied across `OperandContext` with [`copy_operand`](Self::copy_operand).
737///
738/// # Cheap-to-access `Operand`s
739///
740/// In general, creating an `Operand` requires at least a hash table lookup / insertion to
741/// make sure `Operand`s are equal if their references are equal. As such, avoiding
742/// doing repeated hash table lookups by storing the returned `Operand` is preferrable.
743/// However, few of the most common `Operand`s are already cached by `OperandContext`, and
744/// any code having `OperandContext` reference can rely on retreiving these `Operand`s being
745/// a single memory access.
746///
747/// The `Operand`s which are guaranteed to be cached are:
748///
749/// - `OperandType::Arch(n)`, for `n.value() < 16`
750/// - `OperandType::Constant(n)`, for 0, 1, 2, 4, 8, through [`ctx.const_0()`](Self::const_0)
751///     and other functions.
752///
753/// Some code uses this to micro-optimize cases where you want to check if `Operand` equals
754/// to constant zero/one or constant register.
755/// `op.if_constant() == 0` has to do two comparisons and two memory reads:
756/// One to verify `OperandType::Constant` variant, second to verify that the constant is zero.
757/// `op == ctx.const_0()` does just a single memory read to get the cached const_0, and
758/// a single pointer comparison.
759pub struct OperandContext<'e> {
760    next_undefined: Cell<u32>,
761    max_undefined: Cell<u32>,
762    // Contains 0x10 registers, 0x6 flags, SMALL_CONSTANT_COUNT small constants,
763    common_operands: Box<[OperandSelfRef; COMMON_OPERANDS_COUNT]>,
764    // Mutable state that simplify code uses for caching.
765    // Initialized to full of const_0()
766    simplify_cache: [Cell<OperandSelfRef>; SIMPLIFY_CACHE_SIZE],
767    interner: intern::Interner<'e>,
768    const_interner: intern::ConstInterner<'e>,
769    undef_interner: intern::UndefInterner,
770    invariant_lifetime: PhantomData<&'e mut &'e ()>,
771    simplify_temp_stack: SliceStack,
772    // Keep buffer for ExecutionState's Operation::Freeze handling
773    // Unless the user does something weird with intercepting the freeze operations,
774    // this should be the only one that gets ever used
775    // (Semi-ugly code organization for operand to slightly depend on exec_state,
776    // but oh well)
777    freeze_buffer: RefCell<Vec<exec_state::FreezeOperation<'e>>>,
778    // Hashmap of other_op -> own_op, for large copy_operand() calls.
779    // Key is usize, cast from other_op pointer.
780    // Cleared after every copy_operand call.
781    copy_operand_cache: RefCell<FxHashMap<usize, Operand<'e>>>,
782    disasm_cache: RefCell<Option<Box<DisasmCache<'e>>>>,
783    arch_def: &'e ArchDefinition<'e>,
784}

860fn iter_variant_next<'e, T: IterVariant<'e>>(s: &mut T) -> Option<Operand<'e>> {
861    use self::OperandType::*;
862
863    let inner = match s.state() {
864        Some(ref mut s) => s,
865        None => return None,
866    };
867    let next = inner.pos;
868
869    match *next.ty() {
870        Arithmetic(ref arith) | ArithmeticFloat(ref arith, _) => {
871            inner.pos = arith.left;
872            inner.stack.push(arith.right);
873        },
874        Memory(ref m) if T::descend_to_mem_addr() => {
875            inner.pos = m.address().0;
876        }
877        SignExtend(val, _, _) => {
878            inner.pos = val;
879        }
880        Select(a, b, c) => {
881            inner.pos = a;
882            inner.stack.reserve(2);
883            inner.stack.push(b);
884            inner.stack.push(c);
885        }
886        _ => {
887            match inner.stack.pop() {
888                Some(s) => inner.pos = s,
889                _ => {
890                    *s.state() = None;
891                }
892            }
893        }
894    }
895    Some(next)
896}

958    /// Creates a new OperandContext.
959    ///
960    /// This will immediately spend some time interning the most common `Operand`s that will
961    /// be cheap to access later, maybe making this relatively slow operation compared to rest
962    /// of scarf. See [docs on guarantees](./struct.OperandContext.html#cheap-to-access-operands)
963    /// for `Operand`s that are guaranteed to be cheap to access afterwards.

968    pub fn new_with_arch(arch_def: &'e ArchDefinition<'e>) -> OperandContext<'e> {
969        use std::ptr::null_mut;
970        for tag_def in arch_def.tag_definitions.iter() {
971            if tag_def.size == 0 || tag_def.size > 64 {
972                panic!("Invalid arch definition tag size {}", tag_def.size);
973            }
974        }
975        let common_operands =
976            Box::alloc().init([OperandSelfRef(null_mut()); COMMON_OPERANDS_COUNT]);
977        let mut result: OperandContext<'e> = OperandContext {
978            next_undefined: Cell::new(0),
979            max_undefined: Cell::new(0),
980            common_operands,
981            simplify_cache: array_init::array_init(|_| Cell::new(OperandSelfRef(null_mut()))),
982            interner: intern::Interner::new(),
983            const_interner: intern::ConstInterner::new(),
984            undef_interner: intern::UndefInterner::new(),
985            invariant_lifetime: PhantomData,
986            simplify_temp_stack: SliceStack::new(),
987            freeze_buffer: RefCell::new(Vec::new()),
988            copy_operand_cache: RefCell::new(FxHashMap::default()),
989            disasm_cache: RefCell::new(None),
990            arch_def,
991        };
992        let common_operands = &mut result.common_operands;
993        // Accessing interner here would force the invariant lifetime 'e to this stack frame.
994        // Cast the interner reference to arbitrary lifetime to allow returning the result.
995        let interner: &intern::Interner<'_> = unsafe { std::mem::transmute(&result.interner) };
996        let const_interner: &intern::ConstInterner<'_> =
997            unsafe { std::mem::transmute(&result.const_interner) };
998        let base = FIRST_CONSTANT_OP_INDEX;
999        for i in 0..SMALL_CONSTANT_COUNT {
1000            common_operands[base + i] = const_interner.add_uninterned(i as u64).self_ref();
1001        }
1002        let base = FIRST_REGISTER_OP_INDEX;
1003        for i in 0..0x10 {
1004            let arch_id = ArchId {
1005                value: i as u32,
1006                arch_def,
1007            };
1008            common_operands[base + i] =
1009                interner.add_uninterned(&OperandType::Arch(arch_id)).self_ref();
1010        }
1011        let base = FIRST_FLAG_OP_INDEX;
1012        let flags = [
1013            Flag::Zero, Flag::Carry, Flag::Overflow, Flag::Sign, Flag::Parity, Flag::Direction,
1014        ];
1015        for (i, &f) in flags.iter().enumerate() {
1016            let arch_id = ArchId {
1017                value: f as u32 | disasm::make_x86_arch_tag(disasm::X86_FLAG_TAG),
1018                arch_def,
1019            };
1020            common_operands[base + i] =
1021                interner.add_uninterned(&OperandType::Arch(arch_id)).self_ref();
1022        }
1023        let sign_mask_consts = [
1024            0x7fffu64, 0xffff, 0x7fff_ffff, 0xffff_ffff, 0x7fff_ffff_ffff_ffff,
1025            0xffff_ffff_ffff_ffff
1026        ];
1027        let mut pos = SIGN_AND_MASK_NEARBY_CONSTS_FIRST;
1028        for mid in sign_mask_consts {
1029            let count = if mid == 0xffff_ffff_ffff_ffff {
1030                SIGN_AND_MASK_NEARBY_CONSTS / 2
1031            } else {
1032                SIGN_AND_MASK_NEARBY_CONSTS
1033            };
1034            let start = mid.wrapping_sub(SIGN_AND_MASK_NEARBY_CONSTS as u64 / 2).wrapping_add(1);
1035            for val in 0..count {
1036                let n = start.wrapping_add(val as u64);
1037                common_operands[pos] = const_interner.add_uninterned(n as u64).self_ref();
1038                pos += 1;
1039            }
1040        }
1041        let zero = common_operands[FIRST_CONSTANT_OP_INDEX];
1042        for i in 0..SIMPLIFY_CACHE_SIZE {
1043            result.simplify_cache[i].set(zero);
1044        }
1045        result
1046    }

1057    pub fn copy_operand<'other>(&'e self, op: Operand<'other>) -> Operand<'e> {
1058        let self_arch_def = self.arch_def as *const _ as *const ();
1059        let mut ctx = CopyOperandCtx {
1060            arch_def: self_arch_def,
1061            recurse_limit: 32,

1064        if cfg!(debug_assertions) && !std::ptr::eq(ctx.arch_def, self_arch_def) {

1072    fn copy_operand_small<'a: 'e + 'other, 'other>(
1073        &'e self,
1074        op: Operand<'other>,
1075        ctx: &mut CopyOperandCtx,
1076    ) -> Operand<'e> {
1077        if ctx.recurse_limit <= 1 {
1078            let mut map = self.copy_operand_cache.borrow_mut();
1079            if ctx.recurse_limit == 1 {
1080                map.clear();
1081                ctx.recurse_limit = 0;
1082            }
1083            return self.copy_operand_large(op, &mut map, ctx)
1084        }
1085        ctx.recurse_limit -= 1;
1086        let ty = match *op.ty() {
1087            OperandType::Arch(arch) => {
1088                ctx.arch_def = arch.arch_def as *const ArchDefinition<'_> as *const ();
1089                OperandType::Arch(ArchId {
1090                    value: arch.value,
1091                    arch_def: self.arch_def,
1092                })
1093            }
1094            OperandType::Constant(c) => OperandType::Constant(c),
1095            OperandType::Undefined(c) => OperandType::Undefined(c),
1096            OperandType::Custom(c) => OperandType::Custom(c),
1097            OperandType::Memory(ref mem) => OperandType::Memory(MemAccess {
1098                base: self.copy_operand_small(mem.base, ctx),
1099                offset: mem.offset,
1100                size: mem.size,
1101                const_base: mem.const_base,
1102            }),
1103            OperandType::Arithmetic(ref arith) => {
1104                let arith = ArithOperand {
1105                    ty: arith.ty,
1106                    left: self.copy_operand_small(arith.left, ctx),
1107                    right: self.copy_operand_small(arith.right, ctx),
1108                };
1109                OperandType::Arithmetic(arith)
1110            }
1111            OperandType::ArithmeticFloat(ref arith, size) => {
1112                let arith = ArithOperand {
1113                    ty: arith.ty,
1114                    left: self.copy_operand_small(arith.left, ctx),
1115                    right: self.copy_operand_small(arith.right, ctx),
1116                };
1117                OperandType::ArithmeticFloat(arith, size)
1118            }
1119            OperandType::SignExtend(a, b, c) => {
1120                OperandType::SignExtend(self.copy_operand_small(a, ctx), b, c)
1121            }
1122            OperandType::Select(a, b, c) => {
1123                OperandType::Select(
1124                    self.copy_operand_small(a, ctx),
1125                    self.copy_operand_small(b, ctx),
1126                    self.copy_operand_small(c, ctx),
1127                )
1128            }
1129        };
1130        self.intern_any(ty)
1131    }

1133    fn copy_operand_large<'a: 'e + 'other, 'other>(
1134        &'e self,
1135        op: Operand<'other>,
1136        cache: &mut FxHashMap<usize, Operand<'e>>,
1137        ctx: &mut CopyOperandCtx,
1138    ) -> Operand<'e> {
1139        let key = op.0 as *const OperandBase<'other> as usize;
1140        if let Some(&result) = cache.get(&key) {
1141            return result;
1142        }
1143
1144        let ty = match *op.ty() {
1145            OperandType::Arch(arch) => {
1146                ctx.arch_def = arch.arch_def as *const ArchDefinition<'_> as *const ();
1147                OperandType::Arch(ArchId {
1148                    value: arch.value,
1149                    arch_def: self.arch_def,
1150                })
1151            }
1152            OperandType::Constant(c) => OperandType::Constant(c),
1153            OperandType::Undefined(c) => OperandType::Undefined(c),
1154            OperandType::Custom(c) => OperandType::Custom(c),
1155            OperandType::Memory(ref mem) => OperandType::Memory(MemAccess {
1156                base: self.copy_operand_large(mem.base, cache, ctx),
1157                offset: mem.offset,
1158                size: mem.size,
1159                const_base: mem.const_base,
1160            }),
1161            OperandType::Arithmetic(ref arith) => {
1162                let arith = ArithOperand {
1163                    ty: arith.ty,
1164                    left: self.copy_operand_large(arith.left, cache, ctx),
1165                    right: self.copy_operand_large(arith.right, cache, ctx),
1166                };
1167                OperandType::Arithmetic(arith)
1168            }
1169            OperandType::ArithmeticFloat(ref arith, size) => {
1170                let arith = ArithOperand {
1171                    ty: arith.ty,
1172                    left: self.copy_operand_large(arith.left, cache, ctx),
1173                    right: self.copy_operand_large(arith.right, cache, ctx),
1174                };
1175                OperandType::ArithmeticFloat(arith, size)
1176            }
1177            OperandType::SignExtend(a, b, c) => {
1178                OperandType::SignExtend(self.copy_operand_large(a, cache, ctx), b, c)
1179            }
1180            OperandType::Select(a, b, c) => {
1181                OperandType::Select(
1182                    self.copy_operand_large(a, cache, ctx),
1183                    self.copy_operand_large(b, cache, ctx),
1184                    self.copy_operand_large(c, cache, ctx),
1185                )
1186            }
1187        };
1188        let result = self.intern_any(ty);
1189        cache.insert(key, result);
1190        result
1191    }

1195    fn intern(&'e self, ty: &OperandType<'e>) -> Operand<'e> {
1196        debug_assert!(
1197            match ty {
1198                OperandType::Constant(..) | OperandType::Undefined(..) |
1199                    OperandType::Arithmetic(..) => false,

1365    /// Returns [`OperandType::Constant(value)`](OperandType) operand.
1366    pub fn constant(&'e self, value: u64) -> Operand<'e> {
1367        if value < SMALL_CONSTANT_COUNT as u64 {
1368            unsafe { self.common_operands[FIRST_CONSTANT_OP_INDEX + value as usize].cast() }
1369        } else {
1370            // Fast path for values near sign bits or word masks
1371            // 7ff8 ..= 8008
1372            // fff8 ..= 1_0008
1373            // 7fff_fff8 ..= 8000_0008
1374            // ffff_fff8 ..= 1_0000_0008
1375            // 7fff_ffff_ffff_fff8 ..= 8000_0000_0000_0008
1376            // ffff_ffff_ffff_fff8 ..= ffff_ffff_ffff_ffff
1377            let index = ((value as u32 & 0x7fff).wrapping_add(0x8)) & 0x7fff;
1378            if index < SIGN_AND_MASK_NEARBY_CONSTS as u32 {
1379                let rest = value.wrapping_add(8) & 0xffff_ffff_ffff_8000;
1380                if rest.wrapping_sub(1) & rest == 0 {
1381                    let offset = match rest {
1382                        0x8000 => sign_mask_const_index(0),
1383                        0x1_0000 => sign_mask_const_index(1),
1384                        0x8000_0000 => sign_mask_const_index(2),
1385                        0x1_0000_0000 => sign_mask_const_index(3),
1386                        0x8000_0000_0000_0000 => sign_mask_const_index(4),
1387                        0x0 => sign_mask_const_index(5),
1388                        _ => return self.const_interner.intern(value),
1389                    };
1390                    let index = offset.wrapping_add(index as usize);
1391                    return unsafe { self.common_operands[index].cast() };
1392                }
1393            }
1394            self.const_interner.intern(value)
1395        }
1396    }

1512    pub fn arithmetic_right_shift(
1513        &'e self,
1514        left: Operand<'e>,
1515        right: Operand<'e>,
1516        size: MemAccessSize,
1517    ) -> Operand<'e> {
1518        // Arithmetic shift shifts in the value of sign bit,
1519        // that can be represented as bitwise or of
1520        // `not(ffff...ffff << rhs >> rhs)`
1521        let sign_bit = 1u64 << (size.bits() - 1);
1522        let logical_rsh = self.rsh(left, right);
1523        let mask = (sign_bit << 1).wrapping_sub(1);
1524        let negative_shift_in_bits = if let Some(right) = right.if_constant() {
1525            let c = if right >= 64 {
1526                u64::MAX
1527            } else {
1528                (((mask << right) & mask) >> right) ^ mask
1529            };
1530            self.constant(c)
1531        } else {
1532            self.xor_const(
1533                self.rsh(
1534                    self.and_const(
1535                        self.lsh(
1536                            self.constant(mask),
1537                            right,
1538                        ),
1539                        mask,
1540                    ),
1541                    right,
1542                ),
1543                mask,
1544            )
1545        };
1546        let sign_bit_set = self.and_const(left, sign_bit);
1547        // Doing it as `(lhs >> rhs) | select(is_negative, sign_bit_shifted, 0)`
1548        self.or(
1549            self.select(sign_bit_set, negative_shift_in_bits, self.const_0()),
1550            logical_rsh,
1551        )
1552    }

1924    /// Walks through sub-operands of an `Operand`, calling the provided callback
1925    /// that can choose to replace any of them with a new `Operand` by returning `Some()`.
1926    pub fn transform<F>(&'e self, oper: Operand<'e>, depth_limit: usize, mut f: F) -> Operand<'e>
1927    where F: FnMut(Operand<'e>) -> Option<Operand<'e>>
1928    {
1929        self.transform_internal(oper, depth_limit, &mut f)
1930    }

1932    fn transform_internal<F>(
1933        &'e self,
1934        oper: Operand<'e>,
1935        depth_limit: usize,
1936        f: &mut F,
1937    ) -> Operand<'e>
1938    where F: FnMut(Operand<'e>) -> Option<Operand<'e>>
1939    {
1940        if depth_limit == 0 {
1941            return oper;
1942        }
1943        if let Some(val) = f(oper) {
1944            return val;
1945        }
1946        match *oper.ty() {
1947            OperandType::Arithmetic(ref arith) => {
1948                let left = self.transform_internal(arith.left, depth_limit - 1, f);
1949                let right = self.transform_internal(arith.right, depth_limit - 1, f);
1950                if left == arith.left && right == arith.right {
1951                    oper
1952                } else {
1953                    self.arithmetic(arith.ty, left, right)
1954                }
1955            },
1956            OperandType::Memory(ref m) => {
1957                let (base, offset) = m.address();
1958                let new_base = self.transform_internal(base, depth_limit - 1, f);
1959                if base == new_base {
1960                    oper
1961                } else {
1962                    self.mem_any(m.size, new_base, offset)
1963                }
1964            }
1965            OperandType::SignExtend(val, from, to) => {
1966                let new_val = self.transform_internal(val, depth_limit - 1, f);
1967                if val == new_val {
1968                    oper
1969                } else {
1970                    self.sign_extend(new_val, from, to)
1971                }
1972            }
1973            OperandType::Select(a, b, c) => {
1974                let new_a = self.transform_internal(a, depth_limit - 1, f);
1975                let new_b = self.transform_internal(b, depth_limit - 1, f);
1976                let new_c = self.transform_internal(c, depth_limit - 1, f);
1977                if a == new_a && b == new_b && c == new_c {
1978                    oper
1979                } else {
1980                    self.select(a, b, c)
1981                }
1982            }
1983            _ => oper,
1984        }
1985    }

1987    pub fn substitute(
1988        &'e self,
1989        oper: Operand<'e>,
1990        val: Operand<'e>,
1991        with: Operand<'e>,
1992        depth_limit: usize,
1993    ) -> Operand<'e> {
1994        if let Some(mem) = val.if_memory() {
1995            // Transform also Mem16[mem.addr] to with & 0xffff if val is Mem32, etc.
1996            // I guess recursing inside mem.addr doesn't make sense here,
1997            // but didn't give it too much thought.
1998            self.transform(oper, depth_limit, |old| {
1999                old.if_memory()
2000                    .filter(|old| old.address() == mem.address())
2001                    .filter(|old| old.size.bits() <= mem.size.bits())
2002                    .map(|old| {
2003                        if mem.size == old.size || old.size == MemAccessSize::Mem64 {
2004                            with
2005                        } else {
2006                            let mask = match old.size {
2007                                MemAccessSize::Mem64 => unreachable!(),
2008                                MemAccessSize::Mem32 => 0xffff_ffff,
2009                                MemAccessSize::Mem16 => 0xffff,
2010                                MemAccessSize::Mem8 => 0xff,
2011                            };
2012                            self.and_const(with, mask)
2013                        }
2014                    })
2015            })
2016        } else {
2017            self.transform(oper, depth_limit, |old| match old == val {
2018                true => Some(with),
2019                false => None,
2020            })
2021        }
2022    }

2100    /// Converts operand to '32-bit normalized form'
2101    ///
2102    /// This function exists for 32-bit `ExecutionState`, which wants to mainly
2103    /// deal with `Operand`s without caring about their high 32 bits.
2104    ///
2105    /// This could be achieved with just `ctx.and_const(value, 0xffff_ffff)` on every
2106    /// `Operand`, but these masked values would then need to be constantly unwrapped.
2107    /// For values that don't use the entire 32 bits, such as `Mem32[400] << 8`, the
2108    /// mask would become just `0xffff_ff00` after simplification, and determining if
2109    /// the mask is just used intentionally or if it is just to mask out high bits
2110    /// becomes more complicated.
2111    ///
2112    /// So instead of always zeroing high bits, the following kinds of `Operand`s have
2113    /// their '32-bit normalized form' be as is, *without* an bitwise AND mask, even if
2114    /// rest of scarf can consider their high bits to possibly have a meaning. `Operand`s
2115    /// for which scarf can prove low 32 bits be always zero will become zero though, even
2116    /// if they would otherwise be included in the rules below. (E.g. `Mem32[x] << 0x20`)
2117    ///
2118    /// - Any 'pure 64-bit variable'; `Arch`, `Undefined`, `Custom`
2119    /// - Additions, subtractions, consisting only 32-bit normalized
2120    ///     `Operand`s
2121    /// - Multiplications, left shifts consisting 32-bit normalized `Operand` and a constant
2122    /// - *Most XORs* that have both halves 32-bit normalized. (Details left loosely defined
2123    ///     for now though)
2124    /// - `OperandType::ArithmeticFloat`
2125    ///
2126    /// If the `Operand` is `AND 0xffff_ffff` masked, but the value being masked
2127    /// meets above conditions, the mask is removed.
2128    ///
2129    /// If the `Operand` is `AND 0xffff_ffff` masked addition or subtraction, with
2130    /// some of the terms meeting the above conditions, those terms are moved outside
2131    /// of the AND mask. E.g. `(rax + (rcx ^ rdx)) & ffff_ffff` becomes
2132    /// `rax + ((rcx ^ rdx) & ffff_ffff)`.
2133    ///
2134    /// In other cases bitwise AND mask `0xffff_ffff` is applied. (Which does not change the
2135    /// input value if all its high bits are already zero)
2136    ///
2137    /// In the end, it is intended, though not necessarily guaranteed in complex cases,
2138    /// that `ctx.normalize_32bit(x) == ctx.normalize_32bit(x & ffff_ffff)`.
2139    #[inline]
2140    pub fn normalize_32bit(&'e self, value: Operand<'e>) -> Operand<'e> {
2141        if value.0.flags & FLAG_32BIT_NORMALIZED == 0 {
2142            self.normalize_32bit_main(value)
2143        } else {
2144            value
2145        }
2146    }

2148    fn normalize_32bit_main(&'e self, value: Operand<'e>) -> Operand<'e> {
2149        let mut value = value;
2150        if value.relevant_bits().end > 32 {
2151            value = self.and_const(value, 0xffff_ffff);
2152            if value.0.flags & FLAG_32BIT_NORMALIZED != 0 {
2153                return value;
2154            }
2155            // Else value should have removable and now
2156        }
2157        // Only 32-bit value passed here should be and mask
2158        // that can be removed. (Or mask + shift)
2159        if let Some(arith) = value.if_arithmetic_any() {
2160            if value.0.flags & (FLAG_32BIT_NORMALIZED | FLAG_PARTIAL_32BIT_NORMALIZED_ADD)
2161                == FLAG_PARTIAL_32BIT_NORMALIZED_ADD
2162            {
2163                self.normalize_32bit_add(value)
2164            } else {
2165                if arith.ty == ArithOpType::And {
2166                    if arith.left.0.flags &
2167                        (FLAG_32BIT_NORMALIZED | FLAG_PARTIAL_32BIT_NORMALIZED_ADD) ==
2168                            FLAG_PARTIAL_32BIT_NORMALIZED_ADD
2169                    {
2170                        return self.normalize_32bit_add(arith.left);
2171                    }
2172                    return arith.left;
2173                } else if matches!(arith.ty, ArithOpType::Lsh | ArithOpType::Mul) {
2174                    // `(x & mask) << shift` to `x << shift`
2175                    // (Expected that mask can be removed or else it would
2176                    // already be 32bit normalized)
2177                    if let Some(inner_arith) = arith.left.if_arithmetic_any() {
2178                        if let Some(mut shift) = arith.right.if_constant() {
2179                            if arith.ty == ArithOpType::Mul {
2180                                if shift & shift.wrapping_sub(1) != 0 {
2181                                    // Expected to be unreachable but avoid big errors
2182                                    // if there is non-power-of-two mul
2183                                    return value;
2184                                }
2185                                shift = shift.trailing_zeros() as u64;
2186                            }
2187                            if inner_arith.left.0.flags & FLAG_32BIT_NORMALIZED != 0 {
2188                                return self.lsh_const(inner_arith.left, shift);
2189                            }
2190                        }
2191                    }
2192                }
2193                // Expected to be unreachable?
2194                value
2195            }
2196        } else {
2197            // Expected to be unreachable
2198            value
2199        }
2200    }

2202    fn normalize_32bit_add(&'e self, value: Operand<'e>) -> Operand<'e> {
2203        if let Some(arith) = value.if_arithmetic_any() {
2204            // Swap add/sub constants be less than 8000_0000
2205            if let Some(c) = arith.right.if_constant() {
2206                let max = match arith.ty {
2207                    ArithOpType::Add => 0x8000_0000,
2208                    _ => 0x7fff_ffff,
2209                };
2210                if (c as u32) > max {
2211                    let swapped_c = 0u32.wrapping_sub(c as u32) as u64;
2212                    let left = self.normalize_32bit(arith.left);
2213                    if arith.ty == ArithOpType::Add {
2214                        return self.sub_const(left, swapped_c);
2215                    } else {
2216                        return self.add_const(left, swapped_c);
2217                    };
2218                }
2219            }
2220        }
2221        let mut needs_mask = None;
2222        let mut needs_is_sub = false;
2223        let mut doesnt_need_mask = self.const_0();
2224        let mut loop_count = 0;
2225        for _ in util::IterAddSubArithOps::new(value) {
2226            loop_count += 1;
2227            if loop_count >= 16 {
2228                // Too big, just mask the value and consider the work done.
2229                // slow::slow4 test especially ends up hitting edge cases with
2230                // massive (up to ~1000 ops) add/sub chains .
2231                #[cfg(feature = "fuzz")]
2232                tls_simplification_incomplete();
2233                return value;
2234            }
2235        }
2236        for (op, is_sub) in util::IterAddSubArithOps::new(value) {
2237            if op.0.flags & FLAG_32BIT_NORMALIZED == 0 {
2238                needs_mask = match needs_mask {
2239                    None => {
2240                        needs_is_sub = is_sub;
2241                        Some(op)
2242                    }
2243                    Some(old) => {
2244                        if is_sub {
2245                            if needs_is_sub {
2246                                Some(self.add(old, op))
2247                            } else {
2248                                Some(self.sub(old, op))
2249                            }
2250                        } else {
2251                            if needs_is_sub {
2252                                needs_is_sub = false;
2253                                Some(self.sub(op, old))
2254                            } else {
2255                                Some(self.add(old, op))
2256                            }
2257                        }
2258                    }
2259                };
2260            } else {
2261                if is_sub {
2262                    doesnt_need_mask = self.sub(doesnt_need_mask, op);
2263                } else {
2264                    doesnt_need_mask = self.add(doesnt_need_mask, op);
2265                }
2266            }
2267        }
2268        match needs_mask {
2269            Some(needs) => {
2270                let masked = self.and_const(needs, 0xffff_ffff);
2271                if needs_is_sub {
2272                    self.sub(doesnt_need_mask, masked)
2273                } else {
2274                    self.add(doesnt_need_mask, masked)
2275                }
2276            }
2277            None => doesnt_need_mask,
2278        }
2279    }

2310    /// Returns which bits the operand will use at most.
2311    fn calculate_relevant_bits(&self) -> Range<u8> {
2312        match *self {
2313            OperandType::Arithmetic(ref arith) => match arith.ty {
2314                ArithOpType::Equal | ArithOpType::GreaterThan => {
2315                    0..1
2316                }
2317                ArithOpType::Lsh | ArithOpType::Rsh => {
2318                    let left_bits = arith.left.relevant_bits();
2319                    let (min, max) = if let Some(c) = arith.right.if_constant() {
2320                        let c = c as u8 & 0x3f;
2321                        (c, c)
2322                    } else {
2323                        let right_bits = arith.right.relevant_bits();
2324                        if right_bits.end > 6 {
2325                            (0, 0x3f)
2326                        } else {
2327                            (0, (1u8 << right_bits.end).wrapping_sub(1))
2328                        }
2329                    };
2330                    if arith.ty == ArithOpType::Lsh {
2331                        let start = 64u8.min(left_bits.start + min as u8);
2332                        let end = 64u8.min(left_bits.end + max as u8);
2333                        start..end
2334                    } else {
2335                        let start = left_bits.start.saturating_sub(max as u8);
2336                        let end = left_bits.end.saturating_sub(min as u8);
2337                        start..end
2338                    }
2339                }
2340                ArithOpType::And => {
2341                    let rel_left = arith.left.relevant_bits();
2342                    let rel_right = arith.right.relevant_bits();
2343                    max(rel_left.start, rel_right.start)..min(rel_left.end, rel_right.end)
2344                }
2345                ArithOpType::Or | ArithOpType::Xor => {
2346                    let rel_left = arith.left.relevant_bits();
2347                    // Early exit if left uses all bits already
2348                    if rel_left == (0..64) {
2349                        return rel_left;
2350                    }
2351                    let rel_right = arith.right.relevant_bits();
2352                    min(rel_left.start, rel_right.start)..max(rel_left.end, rel_right.end)
2353                }
2354                ArithOpType::Add => {
2355                    // Add will only increase nonzero bits by one at most.
2356                    // And if the bits don't overlap they won't increase even by one.
2357                    // If rhs is constant use bit more accurate bitmask.
2358                    let left_relbits = arith.left.relevant_bits_mask();
2359                    let right_relbits = if let Some(c) = arith.right.if_constant() {
2360                        c
2361                    } else {
2362                        arith.right.relevant_bits_mask()
2363                    };
2364                    let max_value = left_relbits.checked_add(right_relbits)
2365                        .unwrap_or(u64::MAX);
2366
2367                    let rel_left = arith.left.relevant_bits();
2368                    let rel_right = arith.right.relevant_bits();
2369                    let max_end = 64u8 - max_value.leading_zeros() as u8;
2370                    let lower_start = min(rel_left.start, rel_right.start);
2371                    lower_start..max_end
2372                }
2373                ArithOpType::Sub => {
2374                    // Sub will not add any nonzero bits to lower end, but higher end
2375                    // can be completely filled
2376                    let rel_left = arith.left.relevant_bits();
2377                    let rel_right = arith.right.relevant_bits();
2378                    min(rel_left.start, rel_right.start)..64
2379                }
2380                ArithOpType::Mul => {
2381                    let left_bits = arith.left.relevant_bits();
2382                    let right_bits = arith.right.relevant_bits();
2383                    if left_bits.start.wrapping_add(1) == left_bits.end ||
2384                        right_bits.start.wrapping_add(1) == right_bits.end
2385                    {
2386                        // If multiplying by value that has at most 1 bit shift,
2387                        // it is either multiplying by 0 or shifting left,
2388                        // in which case highest nonzero can be known be one less
2389                        // than otherwise.
2390                        // E.g. `x * (y & 0x8000)`
2391                        // Becomes either `0` or `x << 0xf`
2392                        // So result would be (x.start + 0xf, x.end + 0xf)
2393                        //      y relbits are (y.start == 0xf, y.end == 0x10)
2394                        //
2395                        // Instead of having two branches to figure out whether
2396                        // x is on left or right and using `x.end + y.start`,
2397                        // use `x.end + y.end - 1` which works both ways
2398                        let low = left_bits.start.wrapping_add(right_bits.start).min(64);
2399                        let high = left_bits.end
2400                            .wrapping_add(right_bits.end)
2401                            .wrapping_sub(1).min(64);
2402                        low..high
2403                    } else {
2404                        // 64 + 64 cannot overflow
2405                        let low = left_bits.start.wrapping_add(right_bits.start).min(64);
2406                        let high = left_bits.end.wrapping_add(right_bits.end).min(64);
2407                        low..high
2408                    }
2409                }
2410                ArithOpType::Modulo => {
2411                    let left_bits = arith.left.relevant_bits();
2412                    let right_bits = arith.right.relevant_bits();
2413                    // Modulo can only give a result as large as right,
2414                    // though if left is less than right, it only gives
2415                    // left
2416                    if arith.right.if_constant() == Some(0) {
2417                        0..64
2418                    } else {
2419                        0..(min(left_bits.end, right_bits.end))
2420                    }
2421                }
2422                ArithOpType::Div => {
2423                    if arith.right.if_constant() == Some(0) {
2424                        0..64
2425                    } else {
2426                        let left_bits = arith.left.relevant_bits();
2427                        0..left_bits.end
2428                    }
2429                }
2430                ArithOpType::ToFloat => 0..32,
2431                _ => 0..64,
2432            },
2433            OperandType::ArithmeticFloat(ref arith, size) => match arith.ty {
2434                // Arithmetic with f32 inputs is always assumed to be f32 output
2435                // Note that ToInt(f32) => i32 clamped, ToInt(f64) => i64 clamped
2436                // No way to specify ToInt(f32) => i64 other than ToInt(ToDouble(f32)),
2437                // guess that's good enough?
2438                ArithOpType::ToDouble => 0..64,
2439                ArithOpType::ToFloat => 0..32,
2440                ArithOpType::Equal | ArithOpType::GreaterThan => 0..1,
2441                _ => 0..(size.bits() as u8),
2442            }
2443            OperandType::Select(_, a, b) => {
2444                let a_bits = a.relevant_bits();
2445                let b_bits = b.relevant_bits();
2446                min(a_bits.start, b_bits.start)..max(a_bits.end, b_bits.end)
2447            }
2448            OperandType::Arch(arch) => {
2449                let tag = arch.value() >> 16;
2450                if let Some(tag_def) = arch.arch_def.tag_definitions.get(tag as usize) {
2451                    0..tag_def.size
2452                } else {
2453                    0..64
2454                }
2455            }
2456            // Note: constants not handled here; const_relevant_bits instead
2457            _ => 0..(self.expr_size().bits() as u8),
2458        }
2459    }

2471    fn flags(&self, relevant_bits: Range<u8>) -> u8 {
2472        use self::OperandType::*;
2473        // Value of FLAG_32BIT_NORMALIZED for types which are normalized when
2474        // explicitly masked with 0xffff_ffff
2475        let default_32_bit_normalized = if relevant_bits.end > 32 || relevant_bits.start >= 32 {
2476            0
2477        } else {
2478            FLAG_32BIT_NORMALIZED
2479        };
2480        match *self {
2481            Memory(ref mem) => {
2482                let can_simplify_with_and = match mem.size {
2483                    MemAccessSize::Mem8 => 0,
2484                    _ => FLAG_CAN_SIMPLIFY_WITH_AND_MASK,
2485                };
2486                (mem.address().0.0.flags & ALWAYS_INHERITED_FLAGS) | FLAG_NEEDS_RESOLVE |
2487                    FLAG_CONTAINS_MEMORY | default_32_bit_normalized | can_simplify_with_and
2488            }
2489            SignExtend(val, _, _) => {
2490                (val.0.flags & ALWAYS_INHERITED_FLAGS) |
2491                    default_32_bit_normalized | FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2492            }
2493            Select(a, b, c) => {
2494                // Have to always set FLAG_CAN_SIMPLIFY_WITH_AND_MASK since
2495                // one of the inner values may be fully simplifiable to 0 by a mask.
2496                //
2497                // (Or could set it only when inner value relevant_bits aren't 0..0 or 0..64)
2498                ((a.0.flags | b.0.flags | c.0.flags) & ALWAYS_INHERITED_FLAGS) |
2499                    FLAG_CAN_SIMPLIFY_WITH_AND_MASK |
2500                    default_32_bit_normalized
2501            }
2502            Arithmetic(ref arith) => {
2503                let base = (arith.left.0.flags | arith.right.0.flags) & ALWAYS_INHERITED_FLAGS;
2504                let could_remove_const_and = if
2505                    arith.ty == ArithOpType::And && arith.right.if_constant().is_some()
2506                {
2507                    FLAG_COULD_REMOVE_CONST_AND | FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2508                } else {
2509                    match arith.ty {
2510                        ArithOpType::And | ArithOpType::Or | ArithOpType::Xor | ArithOpType::Add |
2511                            ArithOpType::Mul =>
2512                        {
2513                            // If relbits of left/right are not same, simplify_with_and_mask
2514                            // may be able to fully zero one of the operands, so
2515                            // FLAG_CAN_SIMPLIFY_WITH_AND_MASK has to be always set.
2516                            // If they are same, can just set if it either of children have it.
2517                            let child_flags = arith.left.0.flags | arith.right.0.flags;
2518                            match arith.left.relevant_bits() == arith.right.relevant_bits() {
2519                                true => {
2520                                    child_flags & (
2521                                        FLAG_COULD_REMOVE_CONST_AND |
2522                                        FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2523                                    )
2524                                }
2525                                false => {
2526                                    (child_flags & FLAG_COULD_REMOVE_CONST_AND) |
2527                                        FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2528                                }
2529                            }
2530                        }
2531                        ArithOpType::Sub => {
2532                            // If rhs of sub is value with only lowest 1 bit set, it cannot
2533                            // be simplified, because mask passed for sub rhs is 000..111 mask
2534                            // with all 0s below 1 bits set to 1.
2535                            // (At least with current implementation?)
2536                            let right_bits = arith.right.relevant_bits();
2537                            let left_flags = arith.left.0.flags;
2538                            if right_bits.end == 1 {
2539                                (left_flags & FLAG_COULD_REMOVE_CONST_AND) |
2540                                    FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2541                            } else {
2542                                let child_flags = left_flags | arith.right.0.flags;
2543                                (child_flags & FLAG_COULD_REMOVE_CONST_AND) |
2544                                    FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2545                            }
2546                        }
2547                        ArithOpType::Lsh => {
2548                            if arith.right.if_constant().is_none() {
2549                                0
2550                            } else {
2551                                if arith.left.if_memory().is_some() {
2552                                    (arith.left.0.flags & FLAG_COULD_REMOVE_CONST_AND) |
2553                                        FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2554                                } else {
2555                                    arith.left.0.flags &
2556                                        (
2557                                            FLAG_COULD_REMOVE_CONST_AND |
2558                                            FLAG_CAN_SIMPLIFY_WITH_AND_MASK
2559                                        )
2560                                }
2561                            }
2562                        }
2563                        ArithOpType::Rsh => {
2564                            if arith.right.if_constant().is_none() {
2565                                0
2566                            } else {
2567                                arith.left.0.flags &
2568                                    (FLAG_COULD_REMOVE_CONST_AND | FLAG_CAN_SIMPLIFY_WITH_AND_MASK)
2569                            }
2570                        }
2571                        _ => 0,
2572                    }
2573                };
2574                let normalized_32 = if relevant_bits.start >= 32 {
2575                    0
2576                } else {
2577                    Self::arithmetic_32_bit_normalized_flag(arith, default_32_bit_normalized)
2578                };

2580            }
2581            ArithmeticFloat(ref arith, _) => {
2582                (
2583                    (arith.left.0.flags | arith.right.0.flags) & ALWAYS_INHERITED_FLAGS
2584                ) | FLAG_32BIT_NORMALIZED
2585            }
2586            Arch(..) => {
2587                FLAG_NEEDS_RESOLVE | FLAG_32BIT_NORMALIZED
2588            }
2589            Custom(..) => FLAG_32BIT_NORMALIZED,
2590            // Note: undefined not handled here; new_undef instead
2591            // Note: constants not handled here; const_flags instead
2592            Undefined(..) | Constant(..) => 0,
2593        }

2596    fn is_32bit_normalized_for_mul_shift(
2597        op: Operand<'e>,
2598        shift: u8,
2599    ) -> bool {
2600        let ty = op.ty();
2601        if op.relevant_bits().start.wrapping_add(shift) >= 32 {
2602            return false;
2603        }
2604        if let OperandType::Arithmetic(arith) = ty {
2605            match arith.ty {
2606                ArithOpType::Add | ArithOpType::Sub => {
2607                    if !Self::is_32bit_normalized_for_mul_shift(arith.right, shift) {
2608                        return false;
2609                    }
2610                    let mut iter = util::IterAddSubArithOps::new(arith.left);
2611                    if iter.any(|x| {
2612                        !Self::is_32bit_normalized_for_mul_shift(x.0, shift)
2613                    }) {
2614                        return false;
2615                    }
2616                    true
2617                }
2618                ArithOpType::Mul | ArithOpType::Lsh => {
2619                    if let Some(c) = arith.right.if_constant() {
2620                        let shift2 = match arith.ty {
2621                            ArithOpType::Lsh => c as u8,
2622                            _ => c.trailing_zeros() as u8,
2623                        };
2624                        Self::is_32bit_normalized_for_mul_shift(
2625                            arith.left,
2626                            shift.wrapping_add(shift2),
2627                        )
2628                    } else {
2629                        op.is_32bit_normalized()
2630                    }
2631                }
2632                ArithOpType::And => {
2633                    if let Some(c) = arith.right.if_constant() {
2634                        let shifted_c = c << shift;
2635                        // E.g. ((rax & ff) << 18) normalizes to (rax << 18)
2636                        // But any more limiting mask is already normalized.
2637
2638                        // High bits of the shifted mask must be 0,
2639                        // otherwise the normalized form will be one without them.
2640                        let shifted_high_bits_zero = (shifted_c >> 32) as u32 == 0;
2641
2642                        if !shifted_high_bits_zero {
2643                            return false;
2644                        }
2645                        // If shifted mask fills all bits of u32 when shifted in zeroes are
2646                        // changed to ones => nop mask => remove that mask when normalizing
2647                        let nop_mask_if_u32 = (
2648                                shifted_c as u32 |
2649                                    // Any shifted-in bits to 1
2650                                    (1u32.wrapping_shl(shift as u32)).wrapping_sub(1) |
2651                                    // Any bits that are known to be zero in arith.left to 1
2652                                    ((!(arith.left.relevant_bits_mask() as u32))
2653                                        .wrapping_shl(shift as u32))
2654                            ) == u32::MAX;
2655                        if nop_mask_if_u32 {
2656                            // ((rax & ff) << 18) to (rax << 18)
2657                            // but
2658                            // ((rax * rcx) << 18) to (((rax * rcx) & ff) << 18)
2659                            if Self::is_32bit_normalized_for_mul_shift(arith.left, shift) {
2660                                return false;
2661                            }
2662                        }
2663                        return true;
2664                    }
2665                    false
2666                }
2667                _ => {
2668                    // Require other arithmetic always be masked.
2669                    // Technically could keep recursing for or/xor but
2670                    // maybe better to not make this take excessively long?
2671                    false
2672                }
2673            }
2674        } else if let OperandType::Memory(mem) = ty {
2675            let min_shift = match mem.size {
2676                MemAccessSize::Mem8 => 64u8,
2677                MemAccessSize::Mem16 => 24,
2678                MemAccessSize::Mem32 => 16,
2679                MemAccessSize::Mem64 => 0,
2680            };
2681            shift < min_shift
2682        } else if let OperandType::SignExtend(_, from, _) = ty {
2683            let min_shift = match from {
2684                MemAccessSize::Mem8 => 24u8,
2685                MemAccessSize::Mem16 => 16,
2686                MemAccessSize::Mem32 => 64,
2687                MemAccessSize::Mem64 => 64,
2688            };
2689            shift < min_shift
2690        } else {
2691            true
2692        }
2693    }

2695    fn arithmetic_32_bit_normalized_flag(
2696        arith: &ArithOperand<'e>,
2697        default_32_bit_normalized: u8,
2698    ) -> u8 {
2699        match arith.ty {
2700            ArithOpType::And => {
2701                if let Some(c) = arith.right.if_constant() {
2702                    if arith.left.0.flags &
2703                        (FLAG_PARTIAL_32BIT_NORMALIZED_ADD | FLAG_32BIT_NORMALIZED) != 0
2704                    {
2705                        let left_relbits = arith.left.relevant_bits_mask();
2706                        if left_relbits as u32 == c as u32 {
2707                            // mask doesn't change anything for low 32 bits so
2708                            // arith.left is 32bit normalized value of this operand.
2709                            return 0;
2710                        }
2711                    }
2712                }
2713                ((arith.left.0.flags & arith.right.0.flags) & FLAG_32BIT_NORMALIZED) |
2714                    default_32_bit_normalized
2715            }
2716            // Would be nice to have or here too, but (x | y) & c may become
2717            // (x & c) | y which complicates things.
2718            ArithOpType::Xor => {
2719                if (arith.left.0.flags | arith.right.0.flags) & FLAG_COULD_REMOVE_CONST_AND == 0 {
2720                    (arith.left.0.flags & arith.right.0.flags) & FLAG_32BIT_NORMALIZED
2721                } else {
2722                    default_32_bit_normalized
2723                }
2724            }
2725            ArithOpType::Add | ArithOpType::Sub | ArithOpType::Mul |
2726                ArithOpType::Lsh =>
2727            {
2728                if matches!(arith.ty, ArithOpType::Add | ArithOpType::Sub) {
2729                    let mut result =
2730                        arith.left.0.flags & arith.right.0.flags & FLAG_32BIT_NORMALIZED;
2731                    if ((arith.left.0.flags | arith.right.0.flags) & FLAG_32BIT_NORMALIZED) |
2732                        (arith.left.0.flags & FLAG_PARTIAL_32BIT_NORMALIZED_ADD) != 0
2733                    {
2734                        result |= FLAG_PARTIAL_32BIT_NORMALIZED_ADD;
2735                    }
2736                    // Account for (x - 0xffff_ffff) becoming (x + 1) when
2737                    // 32-bit masked
2738                    if let Some(c) = arith.right.if_constant() {
2739                        let limit = match arith.ty {
2740                            ArithOpType::Add => 0x8000_0000u32,
2741                            _ => 0x7fff_ffff,
2742                        };
2743                        if c as u32 > limit {
2744                            return FLAG_PARTIAL_32BIT_NORMALIZED_ADD;
2745                        }
2746                    }
2747                    result
2748                } else {
2749                    let result = arith.left.0.flags & arith.right.0.flags & FLAG_32BIT_NORMALIZED;
2750                    if result != 0 {
2751                        // Mul or Lsh, if lhs is addition / subtraction,
2752                        // and rhs is constant, simplification may prove
2753                        // that some of the terms end up being irrelevant
2754                        // to low 32bits.
2755                        if let Some(c) = arith.right.if_constant() {
2756                            let shift = if arith.ty == ArithOpType::Lsh {
2757                                c as u8
2758                            } else {
2759                                c.trailing_zeros() as u8
2760                            };
2761                            if shift != 0 {
2762                                if !Self::is_32bit_normalized_for_mul_shift(arith.left, shift) {
2763                                    return 0;
2764                                }
2765                            }
2766                        } else {
2767                            // Nonconst mul / shifts are only normalized when less than 32 bit
2768                            // results.
2769                            return default_32_bit_normalized;
2770                        }
2771                    }
2772                    result
2773                }
2774            }
2775            _ => default_32_bit_normalized,
2776        }
2777    }

2794    fn sort_order(&self) -> u64 {
2795        use OperandType::*;
2796        match *self {
2797            Memory(ref mem) => {
2798                let base = mem.address().0;
2799                let base_discriminant = base.ty().discriminant();
2800                ((base_discriminant as u64) << 59) | (base.0.sort_order >> 5)
2801            }
2802            SignExtend(base, _, _) => {
2803                let base_discriminant = base.ty().discriminant();
2804                ((base_discriminant as u64) << 59) | (base.0.sort_order >> 5)
2805            }
2806            Select(a, b, c) => {
2807                // Can this be anything useful? Maybe sort by a/b/c discriminants
2808                let hash = fxhash::hash64(&(a.0.sort_order, b.0.sort_order, c.0.sort_order));
2809                hash
2810            }
2811            Arithmetic(ref arith) | ArithmeticFloat(ref arith, _) => {
2812                let ty = arith.ty as u8;
2813                let hash = fxhash::hash64(&(arith.left.0.sort_order, arith.right.0.sort_order));
2814                ((ty as u64) << 59) | (hash >> 5)
2815            }
2816            // Shift these left so that Arithmetic / Memory sort_order won't
2817            // shift these out. Shifting left by 32 may compile on 32bit host to
2818            // just zeroing the low half without actually doing 64-bit shifts?
2819            Arch(a) => (a.value() as u64) << 32,
2820            Custom(a) => (a as u64) << 32,
2821            // Note: constants / undefined not handled here
2822            Constant(..) | Undefined(..) => 0,
2823        }
2824    }

2826    /// Separate function for comparison when fast path doesn't work
2827    /// (Give compiler chance to inline only fast path if it wants)
2828    ///
2829    /// Assumes that self and other have same `OperandType` variant
2830    /// and that their sort_order was equal
2831    fn cmp_sort_order_was_eq(&self, other: &OperandType<'e>) -> Ordering {
2832        if let OperandType::Memory(ref self_mem) = *self {
2833            if let OperandType::Memory(ref other_mem) = *other {
2834                return self_mem.cmp(other_mem);
2835            }
2836        } else if let OperandType::Arithmetic(ref self_arith) = *self {
2837            if let OperandType::Arithmetic(ref other_arith) = *other {
2838                // Note that ArithOperand (i.e. without sort_order)
2839                // comparison can give different result than OperandType comparison since
2840                // sort_order depends on both left and right, while ArithOperand
2841                // is left-then-right comparison.
2842                return self_arith.cmp(other_arith);
2843            }
2844        } else if let OperandType::ArithmeticFloat(ref self_arith, self_size) = *self {
2845            if let OperandType::ArithmeticFloat(ref other_arith, other_size) = *other {
2846                let order = self_arith.cmp(other_arith);
2847                if order != Ordering::Equal {
2848                    return order;
2849                }
2850                return self_size.cmp(&other_size);
2851            }
2852        } else if let OperandType::SignExtend(inner, from, to) = *self {
2853            if let OperandType::SignExtend(inner2, from2, to2) = *other {
2854                let order = inner.cmp(&inner2);
2855                if order != Ordering::Equal {
2856                    return order;
2857                }
2858                let sizes = ((from as u8) << 4) | (to as u8);
2859                let sizes2 = ((from2 as u8) << 4) | (to2 as u8);
2860                return sizes.cmp(&sizes2);
2861            }
2862        } else if let OperandType::Select(a, b, c) = *self {
2863            if let OperandType::Select(a2, b2, c2) = *other {
2864                return (a, b, c).cmp(&(a2, b2, c2));
2865            }
2866        }
2867        debug_assert!(false, "supposed to be unreachable");
2868        Ordering::Equal
2869    }

3474            let &val = bytes.get(0)?;

3508                    _ => MemAccessSize::Mem64,
3509                };

3511                ctx.sign_extend(inner, from, to)

3513            0x5 => {
3514                use self::ArithOpType::*;
3515                let left = Operand::from_fuzz_bytes(ctx, bytes)?;
